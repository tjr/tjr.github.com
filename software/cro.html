Background Info

Most of what I know about building web applications I have learned from Philip Greenspun. There are hundreds of books and hundreds of thousands of articles about building web applications, but Dr. Greenspun's approach to teaching web software engineering really resonated with me. The foundational element to his pedagogy, as described in Software Engineering for Internet Applications, is for students to build something real, specifically an online community web service of some sort.

My first big web software project was to build an online learning community site for residents of Cedar Rapids, Iowa. I initially built it as a pile of Python scripts run in Apache through mod_python. The system worked, but wasn't very pretty.

I decided to rebuild the system more cleanly as a CherryPy web application. It's still a bunch of Python scripts, but now a little more elegantly designed.

The Big Picture

As a traditional online learning community, the core of this site consists of magnet anchor content in the form of articles on which users can add comments, and a discussion forum. A basic events calendar, consisting of user-contributed event data, is also part of the design.

Implementation

  • The web site is served simply by the built-in CherryPy web server. If this ever proves unable to handle the user load, the CherryPy documentation includes suggestions for other serving options.
  • The data is stored in a half-dozen PostgreSQL tables.
  • The web pages are generated using Python scripts built with the CherryPy web library and the PyGreSQL database library. No HTML templating language is used, as most of the HTML content is generated from data in the database, and I have no problem writing relatively small amounts of HTML directly in Python.
